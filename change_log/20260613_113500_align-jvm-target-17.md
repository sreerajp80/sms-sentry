# Align Java/Kotlin JVM target to 17

Implements `plans/20260613_113500_align-jvm-target-17.md`.

## What changed

`app/build.gradle.kts`:

1. Added import `org.jetbrains.kotlin.gradle.dsl.JvmTarget`.
2. Changed `compileOptions` `sourceCompatibility`/`targetCompatibility` from
   `JavaVersion.VERSION_11` to `JavaVersion.VERSION_17`.
3. Added a `kotlin { compilerOptions { jvmTarget = JvmTarget.JVM_17 } }` block so the
   Kotlin compile task targets JVM 17, matching the Java tasks.

## Why

After `kotlin-android` was applied, `compileDebugKotlin` began running and defaulted its
JVM target to the host JDK (21), while Java was pinned to 11. AGP 8.13 failed the build
with "Inconsistent JVM-target compatibility". Setting both to 17 resolves the mismatch.
