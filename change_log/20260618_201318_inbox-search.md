# Change log: Inbox search facility

Implements [plans/20260618_200211_inbox-search.md](../plans/20260618_200211_inbox-search.md).

## What changed

Added a **scope-aware search** to the Inbox. Single file touched:
`app/src/main/java/in/sreerajp/sms_sentry/ui/SmsOrganizerUi.kt`.

### `InboxScreen`
- Added local search state (`searchQuery`, `trimmedQuery`, `searchActive`) — no ViewModel/DAO/Room
  changes; pure client-side filter over the existing in-memory flows.
- Added `searchSource`: when the active folder is **All**, the source is `inboxMessages +
  spamMessages` (so search reaches Spam); otherwise it is the folder-scoped `filteredMessages`.
- Added `searchResults`: case-insensitive match on **sender OR body**, newest-first, `remember`ed
  on `(searchSource, trimmedQuery)`.
- Added a search `OutlinedTextField` at the top of the inbox (shown when not in selection mode)
  with a leading search icon and a trailing clear button. testTags: `inbox_search_field`,
  `inbox_search_clear`.
- The swipe-hint row is suppressed while searching.
- New render branch: when `searchActive`, show a `LazyColumn` of `SearchResultCard`s, or a
  "No messages match" empty state (scoped wording) when there are no results. When not searching,
  the original conversation-grouped UI is unchanged.

### New composable `SearchResultCard`
- Read-only, tappable message row (avatar, sender, 2-line body snippet, time, category-colour
  border). Tapping calls `viewModel.openThread(msg.sender)`. testTag: `search_result_<id>`.

## Fixes made during implementation
- Search-field padding used incompatible `horizontal` + `top` overload args → switched to
  `start/end/top`.
- The insertion of `SearchResultCard` initially landed between the existing
  `@OptIn(ExperimentalFoundationApi::class)` annotation and `ConversationCard`, breaking its
  opt-in. Moved the annotation back onto `ConversationCard` (`SearchResultCard` only uses the
  stable `clickable`).

## Out of scope (unchanged, per plan)
Top-bar global search from other tabs, searching Accounts/Reminders rows, match highlighting,
search history.

## Verification
- `./gradlew.bat :app:compileDebugKotlin` → **BUILD SUCCESSFUL** (only pre-existing deprecation
  warnings; none from the new code).
- Manual on-device verification (search in All reaches Spam; category folders restrict scope;
  tapping opens the thread; clearing restores the list) not yet run.
