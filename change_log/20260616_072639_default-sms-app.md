# Change log: Make SMS Sentry a real default SMS app (two-way system sync)

Implements [plans/20260614_223309_default-sms-app.md](../plans/20260614_223309_default-sms-app.md)
(all 5 phases, approved by the user).

## Summary

SMS Sentry can now optionally become the device **default SMS app**. When it is, it holds all of
the phone's SMS/MMS, deleting in-app also deletes from the system SMS database, incoming SMS/MMS
are captured live, and sending works via the radio with delivery status. When it is *not* default
it still works read-only (passive listener + import existing SMS), and sending is disabled with an
explanatory message. The inbox is now grouped into per-sender conversations with a chat-style
thread view.

## Known limitations / notes

- **MMS sending is out of scope** (receive/parse/store/display only), per the approved plan.
- **MMS download/parse is carrier- and network-dependent** and hard to test on emulators.
- **Could not compile here** — no Gradle/Android SDK in this environment. Needs a build + on-device
  (ideally emulator first, then a real SIM for MMS) verification pass.
- Dedupe is on `systemId`; a message received via `SMS_RECEIVED` while *not* default has no
  `systemId`, so a later import after becoming default could duplicate it. Acceptable edge case.
- Demo seeding still runs on first launch when `READ_SMS` is not yet granted; real import replaces
  it once permission is granted. Demo rows are not auto-purged.
- Play Store distribution of a default-SMS app needs a Permissions Declaration (not relevant for
  sideload/personal use).
