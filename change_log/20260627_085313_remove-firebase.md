# Remove Firebase from the app

Implements [plans/20260627_085313_remove-firebase.md](../plans/20260627_085313_remove-firebase.md).

## What changed

Removed all vestigial Firebase references left over from the original Google AI Studio export.
Firebase was never used at runtime (no source imports, no `google-services` plugin); only a
version-aligning BOM and a commented-out `firebase-ai` dependency remained.

- **`app/build.gradle.kts`**
  - Removed `implementation(platform(libs.firebase.bom))`.
  - Removed the commented-out `// implementation(libs.firebase.ai)`.

- **`gradle/libs.versions.toml`**
  - Removed the `firebaseBom = "34.12.0"` version.
  - Removed the `firebase-bom` and `firebase-ai` library entries.

- **`docs/architecture.md`**
  - Reworded the "Things worth knowing" note: the Gemini/`firebase-ai` path was never
    implemented, and Firebase has now been removed from the build entirely (the note no longer
    points at a `firebase-ai` line in the build file).

## Verification

- Grep for `firebase` (case-insensitive) across the repo: no remaining build/source references.
  Remaining hits are the architecture-doc note, the plan/change-log files, and
  `.idea/appInsightsSettings.xml` (IDE editor settings, intentionally left out of scope).
- `./gradlew.bat :app:assembleDebug` → **BUILD SUCCESSFUL**, `app-debug.apk` produced.

## Notes

`.env` / `.env.example` `GEMINI_API_KEY` leftovers and the Secrets Gradle plugin were left in
place as agreed (separate cleanup, out of scope).
