# Remove dead code

Implements [plans/20260627_121829_remove-dead-code.md](../plans/20260627_121829_remove-dead-code.md).

## What changed

Removed three declarations that were never referenced anywhere in `app/src/main`, identified by
a full reference scan of declared symbols against their usages:

1. **`SmsOrganizerViewModel.blockSender(msg: SMSMessage)`** — removed the method and its doc
   comment from `app/src/main/java/in/sreerajp/sms_sentry/ui/SmsOrganizerViewModel.kt`. The
   `// --- Block / Report / Delete ... ---` section header was kept (other methods remain under it).
2. **`SmsDao.getKeywordRules()`** — removed the `@Query("... type = 'KEYWORD'")` method from
   `app/src/main/java/in/sreerajp/sms_sentry/data/SmsDao.kt`.
3. **`SmsDao.insertMessages(messages: List<SMSMessage>)`** — removed the batch `@Insert` method
   from the same file.

## Not changed (intentionally)

- `MessageCard` — unused in production but covered by two Robolectric tests; retained.
- Framework overrides (`onBind`, `onStartCommand`) and manifest-wired receivers/services — not dead.
- No Room schema/version change (removing unused DAO methods doesn't alter the schema).
- No imports became unused; the removed code used types still referenced elsewhere.

## Verification

- Confirmed zero remaining references to `blockSender`, `getKeywordRules`, and `insertMessages`
  across `app/src` via grep.
- A full Gradle build was not run as part of this change (no Gradle wrapper in repo). The edits
  are pure deletions of fully-unreferenced declarations, so no call sites are affected.
