# Change log: "Not spam" action for Spam-section conversations

Implements [plans/20260618_082616_not-spam-action.md](../plans/20260618_082616_not-spam-action.md).

## Summary

Added a **"Not spam"** action to the conversation-list selection menu (shown only in the Spam
folder) that moves selected conversations out of Spam by re-classifying them, and allowlists the
sender so future messages are never auto-spammed. Reverse of the existing "Report as spam" flow.

## Changes

### `app/src/main/java/in/sreerajp/sms_sentry/engine/SmsClassifier.kt`
- `classify()` now recognises a `CONTACT -> "NotSpam"` allowlist rule. The custom-contacts loop
  was converted to a `when`: `Spam`/`Blocked` still force Spam, a non-spam category still forces
  that category, and `NotSpam` sets a local `allowlisted` flag and falls through to normal
  classification.
- The spam-heuristic branch is now gated on `!allowlisted`, so an allowlisted sender skips spam
  detection but is still classified into Finance / Reminder / Personal.

### `app/src/main/java/in/sreerajp/sms_sentry/data/SmsRepository.kt`
- Added `restoreFromSpam(message)`: re-classifies the message with Spam/Blocked contact rules
  dropped and the sender forced to `NotSpam`, updates the stored category, unblocks it, and
  rebuilds finance/reminder rows (clearing existing ones first to avoid duplicates) the same way
  `processAndInsertMessage` does.
- Added `allowlistSender(sender)`: deletes any existing `CONTACT` rule(s) whose value matches the
  sender, then inserts a `CONTACT -> "NotSpam"` rule.

### `app/src/main/java/in/sreerajp/sms_sentry/ui/SmsOrganizerViewModel.kt`
- Added `markNotSpamSender(sender)`: allowlists the sender, then calls `restoreFromSpam` for each
  of that sender's messages currently in Spam. Mirror of `reportSpamSender`.

### `app/src/main/java/in/sreerajp/sms_sentry/ui/SmsOrganizerUi.kt`
- Added a "Not spam" `DropdownMenuItem` (icon `Icons.Default.MarkEmailRead`) to the selection
  menu, rendered only when `activeInboxFolder == "Spam"`. On click it calls
  `markNotSpamSender` for each selected sender, shows a "Moved out of spam" toast, and clears the
  selection.

- `"NotSpam"` is only ever a rule `targetCategory`, never a message category, so it does not
