# Remove dead code

**Status:** completed

## Issue

A reference scan across `app/src/main` (matching every declared `fun`/`class`/`object`/
`const val`/`private` member name against all usages in the source tree) surfaced three
declarations that are never referenced anywhere in the codebase. They are pure dead code:

1. `SmsOrganizerViewModel.blockSender(msg: SMSMessage)` — a public ViewModel method that adds a
   `CONTACT->Spam` rule and deletes the message. No call site exists; blocking from the UI goes
   through other paths.
2. `SmsDao.getKeywordRules(): List<FilterRule>` — `@Query` for `type = 'KEYWORD'` rules. Never
   called (the classifier loads rules through other DAO methods).
3. `SmsDao.insertMessages(messages: List<SMSMessage>)` — batch `@Insert`. Never called; the
   repository inserts one message at a time via `insertMessage`.

All other single-reference symbols found by the scan are **not** dead and are explicitly kept:
- `onBind`, `onStartCommand` — Android framework overrides (called by the OS).
- `HeadlessSmsSendService`, all receivers/services — wired via `AndroidManifest.xml`.
- `MessageCard` (in `SmsOrganizerUi.kt`) — unused in production but exercised by two Robolectric
  tests (`ExampleRobolectricTest`) that verify OTP copy-button rendering. Kept, since removing it
  would mean deleting real behavioral tests.

## Files to change

- `app/src/main/java/in/sreerajp/sms_sentry/ui/SmsOrganizerViewModel.kt`
  - Remove the `blockSender` function (and its preceding doc comment).
- `app/src/main/java/in/sreerajp/sms_sentry/data/SmsDao.kt`
  - Remove the `getKeywordRules` query method.
  - Remove the `insertMessages` insert method.

## Plan for the fix

1. Delete `blockSender` (lines ~389–395, including the `/** Block a sender... */` doc comment).
   Keep the `// --- Block / Report / Delete ...` section header only if other methods in that
   section remain; otherwise leave it intact for the methods below it.
2. Delete `getKeywordRules` (its `@Query` annotation + signature, lines ~94–95).
3. Delete `insertMessages` (its `@Insert` annotation + signature, lines ~24–25).
4. No imports become unused as a result (the removed code uses types still referenced elsewhere:
   `FilterRule`, `SMSMessage`, `OnConflictStrategy`).
5. No DB schema/version change is needed — removing unused DAO methods does not alter the Room
   schema, so no migration bump.

## Verification

- Build the app (`docs/build-and-test.md` build command) to confirm it still compiles and the
  existing Robolectric tests still pass.

## Notes

- Scope is intentionally limited to high-confidence, fully-unreferenced declarations. Unused
  parameters, unreachable branches, and unused imports were not pursued (risky / low-value), and
  `MessageCard` is retained because tests cover it.
