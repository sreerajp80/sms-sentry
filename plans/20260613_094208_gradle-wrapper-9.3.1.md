# Plan: Update Gradle wrapper to 9.3.1

**Status:** completed

## Issue

Android Studio (Otter | 2025.2.1) with AGP 9.1.1 reports:

> Minimum supported Gradle version is 9.3.1. Current version is 9.0-milestone-1.

The Gradle wrapper currently points at a pre-release milestone
(`gradle-9.0-milestone-1-bin.zip`), which is below the minimum stable Gradle
version required by the project's Android Gradle Plugin. Project sync/import
fails until the wrapper is bumped to a stable Gradle >= 9.3.1.

## Files to change

- `gradle/wrapper/gradle-wrapper.properties` — the `distributionUrl` line only.

## Fix

Change line 3:

```
distributionUrl=https\://services.gradle.org/distributions/gradle-9.0-milestone-1-bin.zip
```

to:

```
distributionUrl=https\://services.gradle.org/distributions/gradle-9.3.1-bin.zip
```

No other properties change. After the edit, the user re-imports / re-syncs the
project in Android Studio; the wrapper downloads Gradle 9.3.1 on first run.

## Notes / verification

- Only the distribution URL changes; `distributionBase`, `distributionPath`,
  `networkTimeout`, `validateDistributionUrl`, `zipStoreBase`, `zipStorePath`
  stay as-is.
- `validateDistributionUrl=true` means Gradle will verify the URL is reachable
  on first run, so a typo would surface immediately.
- No command-line build is possible to verify here (no global Gradle install);
  verification is done by Android Studio re-sync.
