# Enable `android.useAndroidX` in gradle.properties

**Status:** completed

## Issue

The AGP build fails with:

> Configuration `:app:_agp_internal_javaPreCompileDebug_kspClasspath` contains AndroidX
> dependencies, but the `android.useAndroidX` property is not enabled.

The project pulls in AndroidX libraries (notably `androidx.room:room-compiler:2.7.0` and its
transitive `androidx.annotation`, `androidx.room:*` artifacts) but `gradle.properties` does
not set `android.useAndroidX=true`. AGP refuses to compile in this state.

## Files to change

1. **gradle.properties** — add `android.useAndroidX=true`.

## Plan for the fix

- Append `android.useAndroidX=true` to `gradle.properties`.
- Do NOT add `android.enableJetifier`: all dependencies are already AndroidX, so legacy
  support-library bytecode rewriting is unnecessary (and would only slow builds).

## Notes

- Single-line, low-risk config change explicitly recommended by the AGP error message.
- No Gradle wrapper in this repo (per CLAUDE.md), so a clean compile may not be runnable
  here; correctness verified by the change matching AGP's required property.
