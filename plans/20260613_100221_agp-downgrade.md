# Downgrade AGP 9.1.1 → 8.13.0

## Issue

Gradle sync fails: the project pins the Android Gradle plugin at `9.1.1`, but the
installed IDE tooling supports up to **AGP 8.13.0**.

> The project is using an incompatible version (AGP 9.1.1) of the Android Gradle plugin.
> Latest supported version is AGP 8.13.0

## Files to change

1. **gradle/libs.versions.toml** — `agp = "9.1.1"` → `agp = "8.13.0"`

## Plan for the fix

- Change the single `agp` version literal to `8.13.0` (the latest stable 8.x).
- No change to the Gradle wrapper: it is on Gradle 9.3.1, and AGP 8.13.0 requires
  Gradle ≥ 8.13 and is compatible with the Gradle 9.x line, so it stays as-is.
- No change to Kotlin (2.2.10) or KSP (2.3.5) — both are compatible with AGP 8.13.

## Verification

- Confirm no other reference pins AGP 9.x.
- A full Gradle sync/build confirms the fix if a Gradle environment is available.

## Decided

User chose **Option A** (downgrade AGP) over Option B (keep AGP 9.1.1 and upgrade the IDE).
