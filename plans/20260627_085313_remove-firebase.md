# Remove Firebase from the app

**Status:** completed

## What the issue is

Firebase is vestigial scaffolding left over from the original Google AI Studio export.
It is **not used anywhere**:

- The only active reference is `implementation(platform(libs.firebase.bom))` in
  `app/build.gradle.kts` — a BOM (version-alignment constraint), which adds no code to the APK.
- The only real Firebase artifact, `firebase-ai`, is already commented out.
- No source code touches Firebase (no `com.google.firebase` imports, no `FirebaseApp`,
  no `google-services` plugin or `google-services.json`).
- `docs/architecture.md` already documents that the advertised Gemini/`firebase-ai` path was
  never wired up and classification is 100% offline.

Removing it is dead-config cleanup with no behavioral change, and it removes a
`com.google.firebase` reference that would draw "anti-feature" scrutiny for F-Droid.

## Files to be changed

1. **`app/build.gradle.kts`**
   - Remove line 86: `implementation(platform(libs.firebase.bom))`
   - Remove the commented line 109: `// implementation(libs.firebase.ai)`

2. **`gradle/libs.versions.toml`**
   - Remove the version: `firebaseBom = "34.12.0"` (line 39)
   - Remove the libraries: `firebase-bom = ...` (line 88) and `firebase-ai = ...` (line 89)

3. **`docs/architecture.md`**
   - Update the note at lines 64-67 that references `firebase-ai` being commented out, since
     it will no longer exist. Reword to state the Gemini/LLM path was never implemented and
     Firebase has been removed entirely.

## Plan for the fix

1. Delete the two lines in `app/build.gradle.kts`.
2. Delete the three lines in `gradle/libs.versions.toml`.
3. Reword the architecture doc note so it no longer points at a now-removed dependency.
4. Verify no remaining references: grep for `firebase` (case-insensitive) across the repo;
   the only acceptable remaining hit is the IDE file `.idea/appInsightsSettings.xml`
   (editor settings, not part of the build — leave as-is).
5. Build the release variant to confirm the project still compiles without the BOM.

## Out of scope

- `.env` / `.env.example` `GEMINI_API_KEY` leftovers and the Secrets Gradle plugin are a
  separate cleanup; not touched here unless requested.
