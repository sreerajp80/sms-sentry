# Align Java/Kotlin JVM target to 17

**Status:** completed

## Issue

`gradle :app:compileDebugKotlin` fails with:

```
Inconsistent JVM-target compatibility detected for tasks
'compileDebugJavaWithJavac' (11) and 'compileDebugKotlin' (21).
```

### Root cause

After `kotlin-android` was applied (see `20260613_111819_add-kotlin-android-plugin.md`),
the `compileDebugKotlin` task now actually runs. `app/build.gradle.kts` pins Java
`sourceCompatibility`/`targetCompatibility` to `VERSION_11`, but there is no matching
Kotlin `compilerOptions { jvmTarget }` block, so the Kotlin compiler defaults its JVM
target to the running JDK (21). AGP 8.13 validates that the Java and Kotlin tasks agree
and fails the build when they don't.

## Files to change

1. `app/build.gradle.kts` — set Java compileOptions to 17 and add a Kotlin
   `compilerOptions { jvmTarget = JvmTarget.JVM_17 }` block (with the required import).

## Plan for the fix

1. Add import at top of `app/build.gradle.kts`:
   `import org.jetbrains.kotlin.gradle.dsl.JvmTarget`
2. Change `compileOptions` `sourceCompatibility`/`targetCompatibility` from
   `JavaVersion.VERSION_11` to `JavaVersion.VERSION_17`.
3. Add a top-level (android-sibling) `kotlin { compilerOptions { jvmTarget = JvmTarget.JVM_17 } }`
   block so the Kotlin task targets 17 to match Java.

## Verification

- `gradle :app:compileDebugKotlin` (or `assembleDebug`) completes without the
  "Inconsistent JVM-target compatibility" error.
