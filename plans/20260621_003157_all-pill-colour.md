# Fix: "All" inbox filter pill colour collides with "Personal"

## Issue

In the inbox filter pill row, the **All** pill and the **Personal** pill render in
almost the same colour, so they're hard to tell apart.

Cause: in [SmsOrganizerUi.kt:1205-1211](../app/src/main/java/in/sreerajp/sms_sentry/ui/SmsOrganizerUi.kt#L1205-L1211)
the **All** pill falls through to `MaterialTheme.colorScheme.primary`, while
**Personal** uses `categoryColor("Personal")`. In the default Lavender theme these
are:

- `primary` (All)      = `LavenderPrimaryLight` `#6F57AB` (purple)
- `categoryColor` (Personal) = `CategoryPersonalLight` `#7A59C3` (violet)

These two purples read as the same colour. (Same collision exists in dark mode:
`#8C76C8` vs `#A083EA`.)

## Fix

Give the **All** pill a neutral, theme-independent colour that no category uses,
so it reads as the "everything / no specific category" filter. Use
`MaterialTheme.colorScheme.onSurfaceVariant` (mapped to each scheme's `TextMute`
grey — e.g. Lavender light `#646171`), which is clearly distinct from all four
category colours (violet / amber / teal / red) in both light and dark schemes.

Change the `else` branch only; the four category branches are untouched.

```kotlin
val pillColor = when (folderName) {
    "Personal" -> categoryColor("Personal")
    "Promotions" -> categoryColor("Promotions")
    "Others" -> categoryColor("Others")
    "Spam" -> categoryColor("Spam")
    else -> MaterialTheme.colorScheme.onSurfaceVariant // "All" — neutral, distinct from categories
}
```

## Files to change

- `app/src/main/java/in/sreerajp/sms_sentry/ui/SmsOrganizerUi.kt`
  — line 1210: replace `MaterialTheme.colorScheme.primary` with
  `MaterialTheme.colorScheme.onSurfaceVariant` in the `pillColor` `when`.

## Notes / scope

- No colour-token additions needed; reuses an existing scheme token.
- No tests reference this colour; behaviour (filtering) is unchanged.
- If you'd prefer a coloured "All" pill (e.g. a brand accent that isn't purple)
  instead of a neutral grey, say so and I'll adjust.
