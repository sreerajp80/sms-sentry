# Plan: Add a search facility to the Inbox

## Issue / goal

The app has no way to search messages. The user wants a search facility with **scope-aware**
behaviour:

- When the **All** folder is active → search across **all** messages (every category **plus
  Spam**).
- When a specific folder is active (**Personal / Promotions / Others / Spam**) → search **only
  within that folder/category**.

## Decision: result style

Results are shown **message-level** (one row per matching message), matching against **sender OR
body** (case-insensitive). Tapping a result opens that sender's conversation thread
(`viewModel.openThread(sender)`), consistent with the rest of the inbox. This is more useful than
conversation-level for "find that one text".

## Approach (why this is low-risk)

All inbox data is already held in memory as reactive `StateFlow`s on the ViewModel
(`inboxMessages`, `spamMessages`). Search is therefore a pure **client-side filter over existing
state** — **no Room/DAO/Repository/entity changes**, so no DB version bump and no migration. State
is kept **local to `InboxScreen`** (a `remember`ed query string), matching the app's existing
in-memory / string-switch conventions. No ViewModel changes required.

## Detailed plan

### 1. Search state (local to `InboxScreen`)

Add near the existing `activeInboxFolder` state:

```kotlin
var searchQuery by remember { mutableStateOf("") }
val trimmedQuery = searchQuery.trim()
val searchActive = trimmedQuery.isNotEmpty()
```

### 2. Scope-aware search source + filter

Reuse the existing `filteredMessages` (already respects `activeInboxFolder`) for category scope,
and the union of inbox + spam for "All":

```kotlin
val searchSource =
    if (activeInboxFolder == "All") inboxMessages + spamMessages
    else filteredMessages

val searchResults = remember(searchSource, trimmedQuery) {
    if (trimmedQuery.isEmpty()) emptyList()
    else {
        val q = trimmedQuery.lowercase(Locale.getDefault())
        searchSource
            .filter {
                it.sender.lowercase(Locale.getDefault()).contains(q) ||
                it.body.lowercase(Locale.getDefault()).contains(q)
            }
            .sortedByDescending { it.timestamp }
    }
}
```

Note: "All" currently maps to `inboxMessages` only (excludes Spam); the union above is what makes
"search across all" include Spam as requested.

### 3. Search bar UI

Add a search `TextField` (Material3 `OutlinedTextField`, single line) at the **top of the inbox
content, above the pill-filter `LazyRow`**, shown only when **not** in selection mode (selection
mode already replaces the header). It has:
- a leading `Icons.Default.Search` icon,
- a trailing clear (`Icons.Default.Close`) button that resets `searchQuery` when non-empty,
- placeholder "Search messages", `testTag("inbox_search_field")`.

The pill filters stay visible during search so the active scope is obvious, and switching folders
re-scopes the live results (driven by `searchSource`).

### 4. Render branch

Inside the existing content area, branch on `searchActive`:
- **`searchActive == true`** → render a `LazyColumn` of `SearchResultCard`s from `searchResults`.
  If `searchResults` is empty, show a "No messages match \"<query>\"" empty state (mirroring the
  existing empty-inbox `Box`/`Column` styling). Selection mode and the swipe-hint row are
  suppressed while searching to keep behaviour simple.
- **`searchActive == false`** → existing conversation-grouped behaviour, unchanged.

### 5. New `SearchResultCard` composable

A compact, tappable card (reusing existing helpers: `AvatarTile`, `categoryColor`, the
`SimpleDateFormat` time pattern). Shows the sender, a single-line ellipsised body snippet, the
category-colour accent/border, and time. `onClick = { viewModel.openThread(msg.sender) }`,
`testTag("search_result_${msg.id}")`. No swipe/long-press actions (keeps it read-only and simple).

## Out of scope (not doing unless asked)

- Global search reachable from the top app bar / other tabs.
- Searching Accounts (`FinanceTx`) / Reminders rows.
- Full-text/Room-backed search, highlighting of matched substrings, search history.

## Testing / verification

- Manual: type in the search field in **All** and confirm Spam results appear; switch to a
  category and confirm results are restricted to it; tapping a result opens the thread; clearing
  the query restores the conversation list.
